# umbra — current state (as of 2026-07-21)

## What it is

A shared, installable design-system package (`umbra`, currently `v0.1.0`, `private: true`)
built on `@base-ui/react` + Tailwind CSS 4. Ships theme CSS, shadcn-style primitives, and shared
structural patterns (app shell, settings dialog). Meant to be installed as a real npm dependency
in each consuming app, not copy-pasted — see `README.md` for the full rationale and install/usage
instructions.

Installed from `github.com/JedWag/umbra` via:

```
npm install git+https://github.com/JedWag/umbra.git
```

**Current consumers:** both `qb` (`/home/jed/Eco/qb/frontend/package.json`) and `guzzler`
(`/home/jed/Eco/guzzler/frontend/package.json`) already list it as a dependency.

## Exports (`src/index.ts`)

Everything is re-exported from one barrel: `utils`, `theme-provider`, `theme-toggle`, `app-shell`,
`settings-tab-section`, `settings-dialog-shell`, and all 21 files under `components/ui/`.

## Structural patterns baked into the kit (not just primitives)

- **`AppShell`** (`src/components/app-shell.tsx`) — the standard page shell: `SidebarProvider` +
  supplied `sidebar` + `SidebarInset` with a sticky rounded-corner header (`SidebarTrigger` +
  optional `headerActions`) and a `bg-muted/40` main content area.
- **`SettingsDialogShell`** (`src/components/settings-dialog-shell.tsx`) — full tabbed settings
  dialog: sizing/positioning, tab ribbon (icon + label per tab), and Cancel/Save footer
  (`DialogActions`) all built in. Callers supply `tabs`, `onCancel`, `onSave`.
- **`SettingsTabSection`** (`src/components/settings-tab-section.tsx`) — the title+action-button
  header over a scrollable content area used inside each tab panel.

## Theme (`src/styles/theme.css`)

- Ships both a light and dark palette, both `lab()`/`oklab()` color values extracted from a
  reference dashboard (see `note.md` for the extraction detail and exact values).
- Font: Outfit, loaded via `@fontsource/outfit` (weights 400/500/600) plus a metric-matched
  `Outfit Fallback` `@font-face` to avoid layout shift before the font loads.
- Declares `@source "../**/*.{ts,tsx}"` so Tailwind's content scanner picks up the kit's own
  component source even though it lives in `node_modules` in a consuming project.
- File header explicitly says: do not edit this file in a consuming project — it's meant to stay
  identical everywhere; project-specific overrides go in the project's own CSS, layered after it.

## `docs/DESIGN-SYSTEM.md` sections

Dark Dashboard Design System doc covers (top-level headings, in order): Stack this assumes,
Philosophy, Vocabulary, Typography, Type scale, Palette (dark mode) → Structure/Text/Accent/Status
colors, Full CSS variable table (base-50 → base-1000 ramp + semantic aliases), Spacing system →
Page-title vertical rhythm, Buttons, Table, Settings dialog (tabbed ribbon), Sidebar → the
rounded-corner "floating panel" look, Layout structure, Component conventions to carry over, How
to use this in another project.

## Peer dependencies

Consuming project must provide: `react` ^19, `react-dom` ^19, `tailwindcss`,
`@tailwindcss/vite` (or equivalent), `tw-animate-css`, and `shadcn` (for its base Tailwind layer).
Everything else (`@base-ui/react`, `class-variance-authority`, `clsx`, `cmdk`, `date-fns`,
`lucide-react`, `react-day-picker`, `sonner`, `tailwind-merge`) is a regular dependency and
installs automatically.
